# 用户更新画面

## 1. 基本信息

- 画面ID(路径): /user/update
- 画面名: 用户更新

## 2. 页面概要

- 本页面用于更新已存在用户的基本信息与认证设置。
- 页面显示当前用户的用户主数据，允许修改可变字段并保存。
- 页面字段与用户表定义保持一致，字段类型、长度限制、必填性严格对应表定义。
- 支持密码变更，若填写密码则更新用户密码哈希值。
- 页面展示系统维护字段为只读或隐藏，防止误修改。

## 5. 处理功能详细

### 5.1 初始显示处理 (initializePage)

- 调用时机: 页面首次加载时
- 处理内容:
  - 从 URL 参数获取 userId
  - 调用用户详情获取 API，读取当前用户数据
  - 将用户数据填充到表单字段
  - 将用户ID、用户账号、创建时间、创建者ID、更新时间、更新者ID设置为只读显示
  - 将密码哈希、MFA密钥、逻辑删除标志作为隐藏字段保留
  - 若用户不存在或读取失败，显示错误提示并返回上一页

### 5.2 保存处理 (handleSave)

- 调用时机: 点击保存按钮时
- 处理内容:
  - 对必填字段执行验证：用户显示名、邮箱地址、账户状态
  - 校验邮箱格式和电话号码格式
  - 若登录密码填写，则校验确认密码一致
  - 构建请求数据，调用用户更新 API 保存修改
  - 更新成功后提示成功并返回用户列表页面
  - 更新失败时显示错误提示，保持当前表单状态

### 5.3 取消处理 (handleCancel)

- 调用时机: 点击取消按钮时
- 处理内容:
  - 恢复表单为初始加载时的数据
  - 若存在已修改内容，提示确认放弃修改

### 5.4 返回处理 (handleBack)

- 调用时机: 点击返回按钮时
- 处理内容:
  - 直接返回用户列表页面
  - 不保存当前修改

## 7. 备注・补充

- 页面字段严格依据用户表定义：VARCHAR 长度、BOOLEAN 类型、TIMESTAMP 类型等均保持一致。
- 登录密码为变更输入字段，前端需检查确认密码一致性并提交给后端进行哈希处理。
- 只读展示字段：用户ID、用户账号、创建时间、创建者ID、更新时间、更新者ID。
- 隐藏字段：密码哈希、MFA密钥、逻辑删除标志。
- 邮箱地址和账户状态的长度限制须与用户表定义一致。
