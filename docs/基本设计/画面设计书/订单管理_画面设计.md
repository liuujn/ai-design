# 订单管理画面

## 1. 基本信息

- 画面ID(路径): /order
- 画面名: 订单管理

## 2. 页面概要

- 本页面用于管理订单信息，支持订单的搜索、新建、编辑、详情查看和删除操作。
- 订单列表显示符合条件的订单概要信息，支持分页。
- 点击订单编号可跳转至订单详情/编辑画面。
- 点击新建按钮可跳转至订单创建画面。
- 订单详情画面显示订单头部信息和明细商品列表。
- 订单状态支持流程化操作：确认、发货、完成、取消。

## 5. 处理功能详细

### 5.1 初始显示处理 (initializePage)

- 调用时机: 订单列表页面初次显示时
- 处理内容:
  - 接收URL查询参数
  - 如果URL查询参数中设置了搜索条件，则调用订单列表搜索API获取列表数据
  - 将获取的结果显示在页面上
  - 如果没有搜索结果，显示"未找到符合条件的数据。"
  - 如果获取条数超过1000条，显示提示消息且仅显示前1000条数据
- 备注:
  - 保持搜索结果，分页时不重新执行搜索

### 5.2 搜索处理 (handleSearch)

- 调用时机: 搜索按钮点击时
- 处理内容:
  - 保存搜索条件输入值
  - 组装URL查询并跳转到当前页面 -> 调用初始显示处理

### 5.3 清除条件处理 (handleClear)

- 调用时机: 清除条件按钮点击时
- 处理内容:
  - 将搜索条件恢复为初始值
  - 将列表显示设置为0条

### 5.4 新建跳转处理 (handleCreate)

- 调用时机: 新建按钮点击时
- 处理内容:
  - 跳转到 `/order/create`（编辑画面，空白表单）
  - 模式参数：`mode=create`

### 5.5 详情/编辑跳转处理 (handleDetail)

- 调用时机: 点击订单编号链接或查看图标时
- 输入: 订单ID
- 处理内容:
  - 跳转到 `/order/{orderId}`
  - 查看模式：`mode=view`
  - 若当前用户有编辑权限，显示编辑按钮切换至编辑模式

### 5.6 订单详情/编辑初始显示 (initializeEditPage)

- 调用时机: 订单详情页面加载时
- 处理内容:
  - 从URL参数获取orderId和mode
  - 若为编辑模式（mode=create或mode=edit），表单字段可编辑
  - 若为查看模式（mode=view），所有字段为只读
  - 调用订单详情API获取订单及明细数据
  - 将数据填充到表单字段
  - 根据订单状态动态显示可用的状态操作按钮

### 5.7 保存处理 (handleSave)

- 调用时机: 点击保存按钮时
- 处理内容:
  - 校验必填字段：客户姓名、商品名称、数量、单价
  - 校验数量 >= 1、单价 >= 0
  - 构建请求数据，调用订单创建/更新API
  - 创建模式：POST /api/v1/orders
  - 编辑模式：PUT /api/v1/orders/{orderId}（附带updatedAt）
  - 保存成功后返回订单列表页面
  - 保存失败时显示错误提示，保持当前数据

### 5.8 取消处理 (handleCancel)

- 调用时机: 点击取消按钮时
- 处理内容:
  - 恢复表单为初始加载时的数据
  - 若存在已修改内容，提示确认放弃修改

### 5.9 返回处理 (handleBack)

- 调用时机: 点击返回按钮时
- 处理内容:
  - 直接返回订单列表页面
  - 不保存当前修改

### 5.10 状态变更处理 (handleConfirm / handleShip / handleComplete / handleCancel)

- 调用时机: 点击状态操作按钮时
- 处理内容:
  - 调用订单状态变更API（PATCH /api/v1/orders/{orderId}/status）
  - 请求参数：目标状态、updatedAt（乐观LOCK）
  - 成功后刷新订单数据
  - 失败时显示错误提示
  - 状态流转规则：
    - pending → confirmed / cancelled
    - confirmed → shipped / cancelled
    - shipped → completed
    - completed / cancelled → 不可变更

### 5.11 删除处理 (handleDelete)

- 调用时机: 点击列表中的删除图标时
- 处理内容:
  - 显示删除确认对话框
  - 确认后调用订单删除API（DELETE /api/v1/orders/{orderId}）
  - 请求参数：updatedAt（乐观LOCK）
  - 成功后刷新列表
  - 失败时显示错误提示

### 5.12 明细添加处理 (handleAddItem)

- 调用时机: 点击添加明细按钮时
- 处理内容:
  - 在明细表格末尾添加一条空行
  - 默认值：数量=1，单价=0

### 5.13 明细删除处理 (handleRemoveItem)

- 调用时机: 点击明细删除图标时
- 处理内容:
  - 从明细列表中移除该行
  - 重新计算订单总金额

### 5.14 明细变更处理 (handleItemChange)

- 调用时机: 修改明细行的数量或单价时
- 处理内容:
  - 计算当前行小计：subtotal = quantity * unit_price
  - 重新计算订单总金额：totalAmount = sum(subtotal)
  - 更新页面显示

## 7. 备注・补充

- 页面字段严格依据订单表定义：VARCHAR 长度、DECIMAL 精度、TIMESTAMP 类型等均保持一致。
- 订单明细的小计由前端根据数量和单价自动计算并显示。
- 订单总金额由前端汇总各明细小计实时计算。
- 乐观LOCK使用 updated_at 字段，所有更新/删除/状态变更操作必须携带该字段。
- 订单编号生成规则：ORD + yyyyMMdd + 6位序列号（后端生成）。
- 状态操作按钮根据当前订单状态动态控制显示和启用。
